<h2>WMC (Web Mobile Computing)</h2>
WMC adalah konsep pengembangan sistem yang memungkinkan aplikasi diakses melalui perangkat mobile menggunakan teknologi web maupun aplikasi mobile.<br>
WMC menggabungkan jaringan internet, perangkat mobile, dan aplikasi untuk memberikan layanan yang fleksibel, real-time, dan dapat diakses di mana saja.<br>
Contoh penerapan: e-commerce mobile, sistem transportasi online, dan aplikasi monitoring berbasis web.<br><br>

<h2>Karakteristik WMC</h2>
- Mobile: dapat diakses melalui smartphone/tablet<br>
- Wireless: menggunakan jaringan internet (WiFi/4G/5G)<br>
- Context-aware: dapat menyesuaikan lokasi, waktu, dan kondisi user<br>
- Real-time: data dapat diperbarui langsung<br><br>

<h2>Teknologi Mobile App</h2>
Dalam WMC, terdapat 3 pendekatan utama pengembangan aplikasi:<br><br>

<h2>1. Native App</h2>
Aplikasi dibuat khusus untuk platform tertentu seperti Android (Java/Kotlin) dan iOS (Swift).<br>
Kelebihan:<br>
- Performa sangat cepat dan stabil<br>
- Akses penuh ke hardware (kamera, GPS, NFC, sensor)<br>
- UI/UX lebih optimal<br>
Kekurangan:<br>
- Biaya dan waktu pengembangan lebih besar<br>
- Harus membuat aplikasi terpisah untuk tiap platform<br><br>

<h2>2. Hybrid App</h2>
Aplikasi dibuat menggunakan teknologi web (HTML, CSS, JS) lalu dibungkus ke dalam aplikasi native menggunakan framework seperti Cordova atau React Native.<br>
Kelebihan:<br>
- Satu codebase untuk banyak platform<br>
- Lebih cepat dikembangkan<br>
Kekurangan:<br>
- Performa di bawah native<br>
- Akses hardware terbatas melalui plugin<br><br>

<h2>3. PWA (Progressive Web App)</h2>
Aplikasi web yang dapat berjalan seperti aplikasi mobile tanpa perlu install dari Play Store/App Store.<br>
Kelebihan:<br>
- Ringan dan cepat diakses melalui browser<br>
- Bisa offline (service worker)<br>
- Tidak perlu instalasi<br>
Kekurangan:<br>
- Fitur hardware tidak selengkap native<br>
- Bergantung pada dukungan browser<br><br>

<h2>Perbandingan Singkat</h2>
- Native: performa terbaik, biaya tinggi<br>
- Hybrid: fleksibel, performa menengah<br>
- PWA: paling ringan, mudah diakses<br><br>

<h2>FW PREP (Framework Preparation)</h2>
FW PREP adalah tahap awal sebelum coding aplikasi, bertujuan menyiapkan semua kebutuhan development agar terstruktur dan efisien.<br><br>

<h2>Tahapan FW PREP</h2>
1. Pemilihan Framework<br>
Menentukan teknologi yang digunakan seperti React, Angular, Vue, atau Flutter.<br><br>

2. Setup Environment<br>
Instalasi tools seperti Node.js, NPM, IDE (VS Code), dan dependency lainnya.<br><br>

3. Struktur Project<br>
Membuat folder project, pembagian komponen, dan arsitektur aplikasi (MVC/MVVM).<br><br>

4. Konfigurasi Backend & API<br>
Menyiapkan koneksi ke database, REST API, atau layanan backend seperti Firebase/Supabase.<br><br>

5. Integrasi Library<br>
Menambahkan library tambahan seperti UI framework, authentication, dan state management.<br><br>

<h2>Manfaat FW PREP</h2>
- Mempercepat proses development<br>
- Mengurangi error di awal proyek<br>
- Membuat kode lebih rapi dan terstruktur<br>
- Memudahkan kerja tim<br><br>

<h2>Kesimpulan</h2>
WMC memungkinkan aplikasi berjalan di perangkat mobile dengan berbagai pendekatan (Native, Hybrid, PWA).<br>
Pemilihan teknologi harus disesuaikan dengan kebutuhan aplikasi.<br>
FW PREP sangat penting untuk memastikan proses pengembangan berjalan efektif, cepat, dan terorganisir dengan baik.<br>
